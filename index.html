Cross platform, native mobile apps--in Java

A new approach to cross platform mobile

Why write the same app from scratch for every mobile platform?
Want a way to turn an Android Java codebase into a cross platform codebase, running on Windows and iOS too?

JUniversal makes Java truly cross platform. There are several technologies out there supporting cross platform mobile apps, each with their pros and cons. JUniversal is targetted to Java developers, like those already developing for Android in Java. It uses source code translation. Among the cross platform technologies out there, JUniversal is arguably among the most native-like, as it produces (through translation) code in the native programming language, built/debugged/packaged with native tools. In some ways it's like porting an Android Java app to other platforms but having a computer do the porting rather than a human being.

It's still got lots of rough edges, but the C# translation and JSimple library should work well enough to try out. Check it out. Share what you think.

Here's how it works

1.  You pick the code that you want to share cross platform. It can be a little bit of code (maybe some hairy algorithm) or a lot (everything in your app except the UI). Develop that in Java, with your favorite Java IDE. Ideally, create some JUnit tests for it too. To keep things tidy, separate out shared code to separate modules/JARs.

2.  At build time invoke JUniversal to do source code translation, converting Java to C# (for Windows/Windows Phone) or, coming soon, C++/Objective C++ (for iOS or performance critical code on Android NDK/Windows). JUniversal can also invoke Google's j2objc converter, to produce Objective-C.

Translated code is very human friendly, preserving comments and forwarding. It's intended to look almost like a human being wrote in manually. That's handy when debugging and has some other advantages.

3.  For non-shared parts of your code, write those in the native platform language (C#, Objective-C, etc.). JUniversal doesn't provide any support for UI today--the intention is that you write it natively. That gives the best UX anyway.
